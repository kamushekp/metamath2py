Краткая сводка по задаче (task1)

- Проверено соответствие тулзов из `verification.py` и `database/opensearch_wrapper.py` требованиям агентов из `agent_tasks/main.md`.
- Вывод: набор почти достаточный для G0/A2/A5, но требуются небольшие доработки для удобства и соблюдения ограничений (фиксированная база TRM, без переиндексации).

Что изменено

- database/opensearch_wrapper.py
  - Результаты поиска обогащены позицией фрагмента: `SearchResult` теперь содержит `start_line` и `end_line` (когда доступны).
  - Внутри `search()` добавлено аккуратное извлечение контекста с границами строк через новый приватный метод `_expand_highlight_with_context(...)`. Поведение существующего `_expand_highlight(...)` сохранено для обратной совместимости (оно по‑прежнему возвращает строку; тесты не затронуты).
  - `list_documents(...)` больше не запускает переиндексацию по умолчанию. Добавлен параметр `ensure_index: bool = False`. Это соответствует требованию «индекс корпуса уже существует (не создавать/не обновлять)» у A2.
  - Добавлен вспомогательный метод `get_document_text(rel_path)` для получения полного содержимого файла по относительному пути — удобно для агентов при анализе кандидатов.

- verification.py
  - Исправлена ошибка в `verify_all_proofs(...)`: теперь корректно итерирует имена модулей через `iter_statement_names(root_path)` без несуществующего аргумента.
  - Сигнатуры основных методов не менялись; `verify_proof(...)` уже возвращает структурированный результат и используется `ProofAuthor.verify()` (A5 соответствует).
