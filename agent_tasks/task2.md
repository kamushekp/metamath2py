# Что у нас уже есть:

## 1. Поиск теорем
- Реализован через OpenSearch (файл `opensearch_wrapper.py`).
- Позволяет по текстовому описанию или целевому выражению найти релевантные уже доказанные теоремы из корпуса (~40,000).

## 2. Верификация теорем
- Реализована в файле `verification.py`.
- Проверяет корректность доказательства в Python-представлении (сгенерированного, например, от LLM или вручную написанного).
- Позволяет дать обратную связь о типе ошибки (некорректная подстановка, неправильный шаг и т.д.).

## 3. Цель
- Создавать **новые** теоремы  и доказывать их.
- Агентное взаимодействие должно происходить **поэтапно**: подбор лемм, построение шага, проверка, возможный откат.


# Что нужно сделать:

## Интеграция библиотеки saplings (в папке saplings) с:
- Агентной системой OpenAI Agent SDK (новый формат агентов, поддерживающий memory, tool calling, streaming и т.д.).
- Проектом `metamath2py`, в части использования `opensearch_wrapper.py` и `verification.py`.


# Что предоставляет Saplings
- Фреймворк для реализации деревообразного поиска (tree search) с откатом (backtracking).
- Поддерживает алгоритмы:
  - Depth-First Search (DFS)
  - A* search
  - Greedy Best-First
  - Monte Carlo Tree Search (MCTS)
- Имеет базовую абстракцию `Agent`, которая не совместима с OpenAI Agent SDK.
- Исходный код библиотеки скопирован в проект — возможно модифицировать напрямую.
- MCP-сервер с названием `Context7` подключён, используй эту tool вместе с library /websites/openai_github_io_openai-agents-python for API and docs для того чтобы получать знания о том, что есть в OpenAI Agent SDK.
# Требуется:

1. **Заменить/адаптировать агента Saplings на агент OpenAI SDK**
   - Сохранять состояние агента (context/memory) через механизм OpenAI SDK, или Saplings, или внешнюю структуру.

2. **Реализовать инструменты (`tools`)** для агента:
   - `search_theorems(query: str)` → вызывает `opensearch_wrapper`.
   - `verify_proof(path_to_pyfile: str)` → вызывает `verification.verify_proof()`.

3. **Инкапсулировать дерево поиска (tree of proof attempts)**:
   - Каждая вершина — состояние доказательства (шаги, цель, текущая гипотеза).
   - Дочерние узлы — шаги с различными выбранными теоремами.

4. **Сделать связку с Saplings Search Algorithms**:
  - Например, подключить `AStarAgent` или `GreedyAgent`.

   - предусмотреть возможность переключения алгоритма поиска (не на лету, но при следующем запуске - в качестве конфигурации)
